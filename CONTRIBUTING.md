# Contributing to AVICA

Thank you for your interest in contributing to AVICA!

## Reporting Issues

If you encounter a bug or unexpected behavior, please open an issue and include:

* AVICA version
* Python version
* Operating system
* Steps to reproduce the problem
* Relevant logs or error messages

## Suggesting Features

Feature requests are welcome. Before opening a new issue, please check whether a similar request already exists.

## Development Setup

Clone the repository:

```bash
git clone https://github.com/avikhagol/avica.git
cd avica
```

Create a development environment and install dependencies:

```bash
pip install -e ".[dev]"
```

## Making Changes

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Add or update tests when applicable.
5. Ensure all tests pass.
6. Submit a pull request.

## Code Style

* Follow PEP 8 where practical.
* Write clear docstrings.
* Keep functions focused and well documented.
* Prefer readable code over clever code.

## Pull Requests

Please provide:

* A clear description of the change.
* Motivation for the change.
* Any relevant benchmarks, examples, or screenshots.

Small, focused pull requests are easier to review and merge.

## Questions

If you are unsure about an idea or implementation, feel free to open an issue for discussion before starting work.

Thank you for helping improve AVICA!
